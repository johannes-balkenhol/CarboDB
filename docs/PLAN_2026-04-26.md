## 2. Motifs section: scope clarification + relevance display (MEDIUM)

### Observation
A version from 1-2 weeks ago showed many more motifs (17 EC-keyed motifs like motif_4111, motif_4113b, etc.). Yesterday's fix replaced these with the 7 motifs that 04a_composition.ec_motifs() actually computes (motif_rubisco_kk, motif_rubisco_gk, motif_ca_hh, motif_ca_his_cluster, motif_pepc_rr, motif_biotin_mk, motif_biotin_amk).

### Why the change happened
The 17 old motifs were leftover frontend metadata from before the pipeline was unified. The XGBoost models were trained on the 7-motif vocabulary computed by scripts/04a_composition.py:ec_motifs(). The 17 old motif names were never in the training feature vector — they always rendered as "—" because the API never returned data for those keys. Showing them was misleading.

### What the current 7 motifs mean
These are simple regex patterns probing for residue patterns characteristic of major carboxylase families:
- motif_rubisco_kk, motif_rubisco_gk — paired-residue motifs in RuBisCO active site
- motif_ca_hh, motif_ca_his_cluster — histidine pairs/clusters typical of carbonic anhydrase zinc binding
- motif_pepc_rr — Arg-Arg pair found in PEP carboxylase substrate binding
- motif_biotin_mk, motif_biotin_amk — Met-Lys / Ala-Met-Lys triplets at biotin attachment sites

### Plan
- Surface importance per motif: each motif has a SHAP-derived training importance for both EC and Km models. Add a small "EC weight" + "Km weight" badge to each motif row showing its mean_abs_shap.
- Highlight which motifs match the current sequence: already done (green vs gray rendering). But also rank: matching motifs first, then non-matching; sort within each group by importance for the predicted EC.
- Tooltip showing biological description: already partly there. Make the description visible by default below the motif name (already done) — but also explain what the count means (e.g., "3" = three matches in the sequence) on hover.

### Effort
Small. Mostly UI presentation.

## 3. Standardized labeling (HIGH PRIORITY — TRIVIAL)

### Observation
The page mixes labels like:
- "Predicted Km (CO2)"
- "Km (uM)" in batch results table
- "Experimental Km" in batch results column
- "Your predicted Km" / "Measured Km (CO2)" in nearest-neighbor cards
- "Nearest Match" in batch table
- "Nearest neighbors with experimental Km" as section header

### Decision — adopt these standardized labels everywhere
| Concept | Label |
|---|---|
| The model's Km prediction for the user's input sequence | Predicted Km |
| Experimental Km from BRENDA for the closest BLAST hit | Experimental Km (closest hit) |
| The closest BLAST hit's UniProt ID | Nearest BLAST hit |
| Nearest neighbors that have measured Km in the DB | Nearest hits with experimental Km |

### Effort
Find/replace across AnalysisView.vue and ResultDetail.vue.

## 5. Feature hover tooltips with biological interpretation (MEDIUM)

### Observation
Each SHAP feature is just a name like "Pfam PF02788" or "dipeptide YK" or "ESM-2 dim 1083". Biologists can't infer anything from these without explanation.

### Plan
Native HTML title= tooltip on each feature row, with content that depends on feature type:
- Pfam IDs: accession + human-readable name + e-value/bitscore. e.g. "PF02788 — RuBisCO large subunit, N-terminal domain. Hit in your sequence: e=3.3e-43, bits=147.3"
- Pfam n_hits: "Total number of Pfam domain hits in your sequence (currently {N})"
- Dipeptide: "Frequency of {AA1}-{AA2} dipeptide in sequence. Training Pearson r with log10(Km) = {km_pearson_r}"
- Motif: already in MOTIF_META descriptions
- inv_cat_*: "Catalytic-core feature: density/clustering of D/E/H/K/C/S/T residues"
- ESM-2 dim N: "Protein language model embedding axis {N}. No direct biological meaning, but training showed this dimension correlates with [low/high] Km in EC {ec} enzymes."

### Backend support
Static JSON data/feature_descriptions.json loaded once at frontend startup. For per-prediction context (Pearson r, training mean Km), the API already returns these fields in the SHAP response — verify and surface.

### Effort
Medium. Mostly Vue + a static JSON file.
